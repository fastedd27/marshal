# Marshal: Project Brief

**Client: me.** I run technical operations for a residential smart-home integration company in Central Florida. Every job ends the same way: a tech writes the site document, the as-built record of the install, and hands it to support. Offsite support works the first call; my advanced team takes the escalations.

**What's broken: I am the gate.** Every site doc routes through me, and comes over half-filled: a camera system with no recorder credentials, a control system three rooms use but nobody documented, a remote-access path no one confirmed. So I send it back, chase the tech, recheck. Accountability that should sit with the tech lands on mine. If I make every doc right, I might as well fill them out myself.

**What I've tried:** stricter templates and a second-tech review pass. Neither produced a clean, service-ready report, because nothing reads the answers back and asks the obvious next question. And not every blank is a gap: some sites have no control, no lighting, no AV rooms. Telling a real gap from a system that was never there is the hard part.

**What I need: a reviewer that sits where I sit.** The tech submits a finished doc, and before it is accepted the reviewer decides per system whether support can service it, then routes the exact gaps back to the tech to fix. The rare call a tech cannot make goes to the production manager, not my desk.
